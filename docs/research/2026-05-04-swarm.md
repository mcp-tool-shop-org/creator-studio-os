## Critical correction: Motion's `.motn` format IS documented

Our `docs/reference/motion-automation.md` previously said the format is "undocumented." That was wrong. Apple ships an archived but explicit OZML guide:

- [Motion XML Programming Guide — Customizing](https://developer.apple.com/library/archive/documentation/AppleApplications/Conceptual/motion_XML_guide/Examples/Examples.html)
- [Motion XML Overview](https://developer.apple.com/library/archive/documentation/AppleApplications/Conceptual/motion_XML_guide/Overview/Overview.html)

Key facts:
- Root element is `<ozml version="4.0">`
- Apple-documented external use cases: text replacement, camera-keyframe import, media swap
- Parameter shape: `<parameter name="X" id="1" flags="16" value="...">` with `<curve>` / `<keypoint>` children for animation
- Format is plain-text XML, version-downgradeable by hand-editing the `ozml version` attribute
- **No public open-source `.motn` parser/generator on GitHub as of 2026-05-04**

**Implication:** OZML mutation is a genuinely novel MCP capability. A `motion_template_set_param` tool (open `.motn`, mutate parameter values, save) is feasible *today* with `xmllint` / `fast-xml-parser`. **No prior art exists.** This is the first thing to ship in the next Motion release.

## Per-app gotchas (worth landing in reference docs)

### FCP

- **Compound clips multiply on round-trip.** Effects inside compound clips are routinely lost when round-tripped because FCPXML doesn't carry FCPXMLD bundle internals. Compound authoring is write-mostly; round-trip parity is unsafe. ([forum.blackmagicdesign.com](https://forum.blackmagicdesign.com/viewtopic.php?f=21&t=40534))
- **Connected-clip lane numbering is anchor-host-scoped, not global.** `lane="-1"` is below the primary spine (audio); positive lanes go above (video). Secondary storylines establish their own lane origin — collisions happen if you treat lanes as globally numbered. ([blog.frame.io](https://blog.frame.io/2017/11/30/fcpx-roles/))
- **Roles serialize as dotted strings** (`"dialogue.Mix"`), stored at the clip level, not in a separate `<role>` block. Role-stem export must walk the entire spine.
- **Magnetic Mask is still lost in 1.14** — confirmed unfixed in FCP 12.0 ([fcp.cafe news 2026-04-02](https://fcp.cafe/news/20260402/)).
- **FCPXML 1.14 elements we use are unchanged from 1.13** — the diff is only in smart-collection / match-text / match-analysis-type. Bundling a 1.13 fallback profile is cheap interop with OTIO + Pipeline-Neo + Resolve consumers.

### Compressor

- **Compressor 5.2 silently removed codecs**: H.264 Blu-ray, H.264 interlaced, Dolby Digital, AVC-Intra-on-Apple-Silicon. Old presets that reference these now fail silently. Worth flagging in `compressor_settings_list` results.
- **Watch folders submit partial files.** Sources >~1min copy time get queued before the copy completes. Mitigation: copy to staging, then atomic `mv`. ([macscripter](https://www.macscripter.net/t/watch-folder-for-compressor-droplet-problems/45969))
- **Cluster storage path:** `~/Library/Application Support/Compressor/Storage/<UUID>/{jobs,shared}/` — likely where `compressor_status` can poll queue state without UI scripting. ([discussions.apple.com 5889264](https://discussions.apple.com/thread/5889264))
- **Compressor logs are XML.** Network Encoding Monitor → "save logs" is a better progress source than polling output-file size.
- **`.compressorsetting` plist display name** lives in either a `Name` string or a `NameKey` localization-table reference resolved against the framework's `.lproj` strings. Untested; needs `plutil -p` work.
- **`.compressorbatch` schema is undocumented.** Save-from-GUI → `plutil -p` is the only path to reverse-engineering it.

### Pixelmator Pro

- **Undocumented `with properties` keys for `export`:**
  - WebP: `{compression factor: 0–100, keep transparency: true}`
  - JPEG: `{compression factor, advanced compression: true, keep transparency: true}` — `advanced compression` is Pixelmator's mozjpeg path, NOT in the public sdef prose
  - Discovered via [forum.latenightsw.com](https://forum.latenightsw.com/t/i-made-a-pixelmator-pro-library/2712) Late Night Software AppleScript library
- **Top-level commands easy to miss:** `detect face`, `detect QR code`, `replace text` are top-level, not layer methods.
- **Color adjustments + layer styles are layer *properties*, not contained child objects** — different from Photoshop's mental model.
- **Shortcuts.app exposes 28 actions** that are partially missing from the sdef: `Increase Resolution of Image`, `Match Colors of Images`, `Optimize Image for Web`, `Overlay Image`, `Remove Background from Portrait Photo`, `Replace Layer in Document`, `Replace Text in Document`. Some ML knobs are Shortcuts-only — worth a `pixelmator_run_shortcut` bridge.
- **Standalone Pixelmator Pro 3.8 vs Creator Studio 4.2 share `com.apple.pixelmator` bundle ID** but both can coexist — disambiguate by version string.

### Logic Pro

- **Logic 12 (Jan 2026) added zero programmatic surface.** AI Session Synth Player and Chord ID are user-facing. ([release notes](https://support.apple.com/en-us/109503))
- **No `logicpro://` URL scheme.** No deeplink-to-project mechanism.
- **OSC is the only first-party programmatic surface** and it's locked to control-surface semantics — UDP/IPv4, predefined paths, mostly one-way. ([Apple OSC paths](https://support.apple.com/guide/logicpro/osc-message-paths-ctlsf67f4bdc/mac))
- **`com.apple.mobilelogic` bundle ID** likely tracks Apple's internal "mobilelogic" engine codename when iPad Logic Pro's shared core was unified into Mac Logic. Both apps run the same engine now.
- **`.logicx`'s `ProjectData` binary is undocumented and version-volatile.** Reversing it is EULA-adjacent. Don't.
- **The plists in `Alternatives/*/Metadata/`, `ProjectInformation`, etc. ARE readable** — sample rate, tempo, length, track count, plugin manifest. ~80% of "what's in this session" answers without touching the binary.
- **Scripter is a JavaScriptCore sandbox** — no FS, no sockets, no `require`. External bridges are MIDI-out → IAC bus → external process patterns.
- **The two existing Logic MCPs (`koltyj`, `che-logic-pro-mcp`) are keystroke macros** dressed up. They don't unlock any surface we don't already have via `logic_open` — they just script the menu bar.

### Motion

- **OZML 4.0 is documented** — see correction above.
- **No `motion -render` CLI.** Confirmed across community threads from 2008 to 2026.
- **`.motn` is plain XML.** Hand-edited by community for years for text/media/parameter swap.
- **The render path is still `cmd-E` Send to Compressor** (UI scripting) or human-mediated. After Effects + `aerender` is the cleaner choice for unattended render; Motion is fundamentally GUI-bound for render.

### Keynote / Pages / Numbers

- **`make new slide` is the spine for Keynote authoring**, with `at` positional variants. Returns a slide reference. Theme swapping on existing documents IS supported despite sparse Apple docs. ([iworkautomation.com](https://iworkautomation.com/keynote/slide-make.html))
- **Keynote shape text uses `object text:` key**, not `text:` — distinct from Pages.
- **Default shapes spawn very small** — set width/height in same `with properties` record or they're invisible.
- **Font >48pt clipping bug** in Keynote: create text box, *resize first*, then set text.
- **Keynote `password` export property silently no-ops on PNG/JPEG** — only valid for PDF.
- **Pages `body text` is a property, not a class.** Page-layout documents have no default text flow; writes silently fail. Always gate on `document body` boolean.
- **Pages `every word of body text where ...` hangs.** Always traverse `every paragraph` first.
- **Pages mail merge (12.1+) requires Numbers source, not CSV.** Letters/cards/envelopes only — no labels, no multi-record-per-page. ([TidBITS](https://tidbits.com/2022/06/30/apple-brings-mail-merge-back-to-pages/))
- **Pages image insertion is the universal weak point.** `make new image with properties {file:, position:, width:, height:}` works on some versions; others require pasteboard insertion with unpredictable placement.
- **Numbers formulas are write-only.** Setting `value` to `"=SUM(B2:B10)"` works; reading back returns the computed result, not the formula text. Track formulas in your own state.
- **Numbers has NO chart sdef.** Chart styling (titles, labels, colors) is GUI-scripting only. Worth documenting as a hard limitation.
- **`formatted value` vs `value`:** `value` is the typed primitive (number/text/date/bool); `formatted value` is the displayed string (currency, separators). Don't conflate.
- **Numbers Excel export degrades silently.** Numbers-specific functions (DURATION, RAWVALUE, regex flavor in TEXTBETWEEN) don't survive. Charts round-trip as static images.
- **osascript startup ~400ms** — compound tools are mandatory for batch work in iWork. Reichenbach bundles compound ops (`numbers_create_sheet_with_table`).
- **Late Night Software Pixelmator AppleScript library** ([forum.latenightsw.com](https://forum.latenightsw.com/t/i-made-a-pixelmator-pro-library/2712)) ships 5 suites with 27+ blend modes — port the enum.

## Priority next-step recommendations (synthesized)

Order by leverage and novelty:

1. **`motion_template_set_param`** — OZML 4.0 mutation. Open `.motn`, mutate `<parameter value="...">`, save. **Zero prior art globally** — first-mover advantage. Pairs with FCP `<title>` `<param>` overrides for the canonical Motion-template-from-FCPXML pipeline.
2. **`compressor_settings_inspect`** — `plutil -convert xml1` + parse for `Name` / `NameKey`. Doubles as `.compressorbatch` schema-reverse prep for v1.3.
3. **Keynote authoring wing** — `keynote_make_slide`, `_set_slide_master`, `_insert_text/image/shape`, `_apply_theme`, `_from_markdown`. Closes the gap to `iwork_mcp`'s 41 tools. md2key-style master mapping (`cover`/`h1`/`h2`/...) is high-leverage.
4. **`numbers_write_table` (2D bulk) + `numbers_set_formula`** — sdef-supported, cross-sheet refs work today. Match `iwork_mcp`'s baseline.
5. **`numbers_file_*` headless lane via [`numbers-parser`](https://pypi.org/project/numbers-parser/)** — read/write `.numbers` without launching the app. Wins on speed + headless CI for batch report generation.
6. **`pixelmator_make_layer` + `pixelmator_apply_ml`** — author from scratch (text + shape layers); expose ML algorithm enum (`ml super resolution`, `ml denoise`, `ml match colors`, `ml remove background`). Plus `pixelmator_run_shortcut` bridge for ML knobs only Shortcuts exposes.
7. **`pages_new_from_markdown`** — Pandoc → DOCX → Pages open. Skips the brittle body-text walker entirely.
8. **`logic_project_inspect`** — parse plists in `.logicx/Alternatives/*/Metadata/`, `ProjectInformation/`, etc. Skip the binary `ProjectData`. Returns sample rate, tempo, length, track count, plugin manifest. ~80% of "what's in this session" without touching reverse-engineered formats.
9. **`fcp_validate_compound_safety`** — pre-flight tool that flags spec compounds that won't survive round-trip.
10. **`fcp_targetVersion: "1.13" | "1.14"`** — cheap interop with OTIO + Pipeline-Neo + Resolve consumers. DareDev256 caps at 1.11 so 1.13 is the broader interop default.
